Prueba cero: Creación del mundo

Borrar clase prueba y crear una nueva.

Nombrar la nueva clase MainGame y heredar de la clase Game. Dar doble clic en el recuadro que aparece abajo para importar su paquete.

Presionar alt+intro y dar clic en implement methods, dar clic en OK en el siguiente menu.

Ir al DesktopLauncher y cambiar prueba por MainGame, dar doble clic en el recuadro de abajo.

Crear pantalla base

Crear nueva clase en donde se encuentra MainGame, implementar la clase Screen, dar doble clic en el recuadro para importar su paquete. Después presionar alt+intro para implementar sus métodos

Se crea un objeto de tipo MainGame y se crea el constructor de BaseScreen con un objeto de MainGame. Esto se realiza para conectar las pantallas que heredarán de BaseScreen a MainGame

Mostrar pantalla de físicas

Se crea nueva clase Box2dScreen la cual hereda de BaseScreen, presionar alt+intro para crear el constructor

Crear las funciones sho, dispose y render. Cuando se escribe la función, presionar intro para crear autocompletar la función. Borrar las líneas super de cada función que se creó.

En la función de render, limpiar la pantalla usando la siguiente función, mientras se escribe, presionar alt+intro para importar su paquete.

```javascript Gdx.gl.glClear(GL20.GL_COLOR_BUFFER_BIT); ```

Después ir a MainGame para crear la nueva pantalla.

Crear el mundo de físicas

Crear variables de World, Box2DDebugRenderer y OrthographicCamera. Importar sus paquetes.

Usar el método dispose en la función dispose para borrarlos cuando la aplicación se cierre.

Inicializar las 3 variables en la función show

World: Permite crear la dirección de la gravedad, y con true le decimos a Box2D que deje de simular cuando no haya algo en movimiento.

Box2DDebugRenderer: Sirve para ver los objetos.

OrthographicCamera: Sirve para darle tamaño a la pantalla en metros.

Por ultimo, escribir las siguientes líneas para terminar de crear el mundo.

World.step: Sirve para definir el cada cuando se realizará los cálculos Box2D, la documentación recomienda (6, 2).

El valor delta viene dado por los FPS.

Camera.update: Sirve para actualizar lo que se ve en pantalla.

Renderer.render: Sirve para decir que dibuje a World con la cámara ortográfica
